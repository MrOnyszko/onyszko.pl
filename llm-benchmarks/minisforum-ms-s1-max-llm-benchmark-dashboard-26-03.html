🔑 Key Finding: MoE vs Dense Architecture

MoE (Mixture-of-Experts) models like qwen3.5-35b-a3b and qwen3.5-122b-a10b maintain near-constant generation speed across all context lengths because only active parameters scale with context. Dense models like qwen3.5-27b degrade severely — TTFT at 75% fill reaches 707–784 seconds, making them impractical beyond ~25k tokens on this hardware.

⚠ The TTFT Paradox at 90–95% Fill

Counterintuitively, TTFT drops at 90–95% fill vs 75% fill for most models. This is a KV cache paging / chunked prefill artifact: at very high fill ratios, the scheduler processes fewer new tokens per step, so individual step latency looks lower even though the total work is similar. Don't be fooled — absolute latency is still high.
